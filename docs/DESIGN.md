# Design

EasyFrontEnd should feel like a clear, work-focused visual builder rather than a blank design canvas. The product is for beginners who need confidence, not limitless controls.

## Interaction Principles

- Name sections by business purpose, not implementation detail.
- Prefer direct manipulation: select, edit, reorder, duplicate, hide, export.
- Keep controls semantic: tone, density, variant, emphasis, CTA text.
- Hide implementation details unless an advanced mode explicitly asks for them.

## Visual Tone

The MVP UI should be calm, structured, and readable. It should avoid decorative distractions and emphasize the canvas, section structure, and inspector feedback.
